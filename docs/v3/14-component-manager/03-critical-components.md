# Critical Components

> [!NOTE]
> **Required role:** `component_manager`
> **Required account type:** Corporate, Enterprise or Medical

The Critical Components view shows all components flagged as critical across the entire company portfolio. It provides a single dashboard for component managers to monitor high-risk dependencies.

## What you see

- Components flagged as critical in any module across the company.
- Aggregate statistics: how many projects and modules use each critical component.
- Vulnerability status and license compliance for each critical component.
- Version distribution across the portfolio.

📸 *Screenshot: the Critical Components dashboard with flagged components, usage counts and vulnerability status.*

## Typical workflow

1. Review the list of critical components regularly.
2. Check for new vulnerabilities or EOL announcements.
3. Coordinate replacement or upgrade across affected projects.
4. Track progress via the component's detail view.
