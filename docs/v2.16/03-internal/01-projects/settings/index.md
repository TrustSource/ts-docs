# Project Settings

Twelve tabs of project-level configuration. Tab visibility depends on your role and on the resource-based field-level permissions.

## In this chapter

| Page | Status | Scope |
|---|---|---|
| [General](01-general.md) | _Scaffold_ | Name, description, tags, default project / compliance manager, members. |
| [Legal](02-legal.md) | _Scaffold_ | Project-level legal settings, wizard, template import, propagate to modules. |
| [CIA](03-cia.md) | _Scaffold_ | Confidentiality / Integrity / Availability requirements at project level. |
| [Quality](04-quality.md) | _Scaffold_ | Quality settings (e.g. coverage requirements) at project level. |
| [Encryption](05-encryption.md) | _Scaffold_ | Project-level encryption requirements. |
| [End of Life](06-end-of-life.md) | _Scaffold_ | EOL thresholds and notification rules at project level. |
| [Whitelists & Blacklists](07-whitelists-blacklists.md) | _Scaffold_ | Per-project allowed / forbidden licenses and components, enforce flag. |
| [Obligations](08-obligations.md) | _Scaffold_ | License obligations configuration; import; propagate to modules. |
| [Module Classification](09-module-classification.md) | _Scaffold_ | MDR / IVDR settings at project level. |
| [Issue Tracker](10-issue-tracker.md) | _Scaffold_ | Jira / TFS / GitHub project-level issue creation configuration. |
| [Module Import](11-module-import.md) | _Scaffold_ | Bulk-import modules from CSV / JSON. |
| [Danger Zone](12-danger-zone.md) | _Scaffold_ | Project deletion (PM-as-default-owner only). |
