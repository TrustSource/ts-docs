# Project Settings

Project settings are organized in twelve tabs. Tab visibility depends on your role. Settings defined here act as **defaults** for all modules in the project — modules can override most values.

| Tab | What it configures |
|---|---|
| [General](01-general.md) | Name, description, tags, default managers, members, confidence level. |
| [Legal](02-legal.md) | Commercial aspects, operating model, IP protection flags. |
| [CIA](03-cia.md) | Confidentiality, Integrity, Availability requirement levels. |
| [Quality](04-quality.md) | Test and quality requirements. |
| [Encryption](05-encryption.md) | Cryptographic algorithm and export control settings. |
| [End of Life](06-end-of-life.md) | EOL analysis toggle, warning and violation thresholds. |
| [Whitelists & Blacklists](07-whitelists-blacklists.md) | Allowed/denied licenses and components, enforce flag. |
| [Obligations](08-obligations.md) | License obligation configuration, import/propagate. |
| [Module Classification](09-module-classification.md) | MDR/IVDR classification (Medical accounts). |
| [Issue Tracker](10-issue-tracker.md) | Jira, GitHub or TFS integration per project. |
| [Module Import](11-module-import.md) | Bulk-import modules from files. |
| [Danger Zone](12-danger-zone.md) | Delete the project. |

> [!TIP]
> Many tabs offer **propagate to modules** — push the current project-level value to all modules at once.
